## Project Overview
This project documents the design, implementation, and validation of an enterprise-style network in a home lab environment, alongside a Cisco Packet Tracer simulation used for learning and documentation purposes. The lab is designed to reinforce core networking concepts while serving as a long-term technical portfolio that demonstrates practical networking skills.

Version 1 focuses on establishing a stable and functional LAN environment that includes routing, switching, wireless access, and basic VoIP services. The project is documented incrementally through structured milestones. Each milestone represents a stable checkpoint rather than a continuous stream of changes and includes design intent, implementation details, validation procedures, and lessons learned.

---

## Project Objectives
The primary objectives of this project are to:
* Design and implement a functional routed and switched network.
* Establish basic Internet edge connectivity.
* Integrate wireless networking and VoIP services.
* Document and validate a baseline infrastructure that supports iterative expansion into advanced networking concepts and multivendor implementations.

---

## Design Scope
Version 1 implements a collapsed-core design focused on foundational connectivity and services, and intentionally accepts single points of failure: 
* Single router performing all Layer 3 routing.
* Single access switch.
* Single 802.1Q trunk link.

High availability, redundancy, dynamic routing, centralized services, and security hardening are intentionally deferred to later versions.

The scope includes:
* Access-layer switching.
* Router-on-a-stick inter-VLAN routing.
* VLAN-based network segmentation.
* A single 802.1Q trunk link.
* DHCP and foundational IP services.
* Internet edge routing with NAT.
* Wireless LAN integration.
* VoIP phone registration using DHCP Option 150.

## Network Design
This section defines the physical and simulated platforms used to implement and validate Version 1 of the network.

### Tools & Platforms
The physical and simulated environments are maintained in close alignment to ensure configuration consistency and repeatable validation.

#### Hardware
* Cisco 2811 Integrated Services Router (ISR). Provides:
  * Router-on-a-stick inter-VLAN routing.
  * Internet edge routing with NAT.
  * DHCP service.
  * Voice service.
* Cisco 2960 Catalyst switch. Provides:
  * Layer 2 endpoint connectivity.
* Cisco 1921 ISR (future edge migration).
* Cisco 3560 Catalyst multi-layer switch (future distribution layer).
* Cisco 8861 IP Phone.

#### Software
* Cisco Packet Tracer for network simulation.
  * The physical lab topology is mirrored in Packet Tracer to validate configurations prior to deployment and provide downloadable labs that support documentation. Packet Tracer complements, but does not replace, hands-on hardware experience.

### VLAN Segmentation
VLAN segmentation is used to logically separate traffic types and establish foundational Layer 2 boundaries within the LAN.

| VLAN ID | Name | Purpose |
| :---: | :---: | :---: |
| 10 | DATA | End-user wired and wireless clients |
| 20 | VOICE | VoIP services |
| 99 | MGMT | In-band device management |
| 88 | NATIVE | Native VLAN for 802.1Q trunks (no user traffic) |
| 404 | INACTIVE | Unused switch ports (administratively shutdown) |

### IP Addressing
* Internal LAN networks use private IPv4 addressing from the 192.168.0.0/16 range.
  * Subnets are sized conservatively to accommodate future growth within each VLAN.
* WAN and point-to-point links use the 203.0.113.0/24 RFC 5737 documentation network.
  * This prevents overlap with real public address space.
* Router sub-interfaces serve as default gateways.
* DHCP Option 150 will be used for VoIP TFTP provisioning.

## Versioning & Future Growth
This project follows a versioned implementation model to promote stability and reinforce structured learning. Each version introduces new technologies or architectural changes while preserving a documented baseline for comparison and iterative improvement.

Planned future versions will migrate toward a hierarchical three-tier model which include:
* Migration to multilayer switching.
* Dedicated edge routing.
* Dynamic routing protocols.
* Redundancy and high availability.
* Centralized network services.
* Layer 2 and Layer 3 security controls.
* Multivendor network implementations using EVE-NG or GNS3.

---

## Documentation Approach
Each major implementation phase is documented as a milestone post. This structured approach creates a clear record of technical progress and design decision-making over time.

Milestone documentation includes:
* Design intent.
* Implementation details.
* Validation and testing procedures.
* Lessons learned.

This methodology emphasizes repeatability, validation, and measurable technical growth.

---

## Closing Notes
By beginning with a simple architecture and layering complexity over time, the lab remains both educational and sustainable. The goal is not complexity, but demonstrable understanding, intentional design decisions, and measurable technical growth.
