Linux File System & Permissions

  • 12/9/2025
  • 2-minute read

Howdy! Thanks for stopping by. I’m Moses and I’m a Network Engineer for an ISP.

Linux File System

Location Description
/ Primary hierarchy and root directory of the entire file system.
/bin System boot loader files.
/boot System boot loader files.
/dev Device files.
/etc Host-specific configuration files.
/home User home directory.
/lib Shared library modules.
/media Media files such as CD-ROM.
/mnt Temporary mounted file systems.
/opt Add-on application packages.
/proc Automatically generated files
/root Home directory for the root user.
/run Run time program data.
/sbin System binaries.
/srv Site-specific data served on system.
/sys Virtual directory info of system.
/tmp Temporary files.
/usr Ready-only user files.
/var Temporary files.


Linux File Permissions

U = User
G = Group
W = World

r = Read
w = Write
x = Execute
- = No permissions

4 = read (r)
2 = write (w)
1 = execute (x)

The chmod command is used to modify file permissions.

User Permissions Group Permissions World Permissions Command Result
rwx rwx rwx chmod 777 read, write, and execute (rwx) for all
rwx rwx r-x chmod 775 read, write, and execute for owner and group; read and execute for world
rwx r-x r-x chmod 755 read, write, and execute for owner; read and execute for group and world
rw- rw- r-- chmod 664 read and write for owner and group; read only for world.
rw- r-- r-- chmod 644 read and write for owner; read only for group and world.
